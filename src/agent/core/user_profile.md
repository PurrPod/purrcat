## 合规性验证认知
### 🔍 "基于"的深层含义
1. **表面理解**：创作时参考了前置输出
2. **实际要求**：在交付物中**显式展示**这种关联，以供验证
3. **验证标准**：必须提供可验证的依赖证明

### ⚖️ 质量与合规性平衡
1. **质量维度**：交付物本身的内容完整性和专业性
2. **合规性维度**：符合任务描述中的所有约束条件
3. **平衡原则**：两者同等重要，缺一不可
4. **失败教训**：质量好但不符合约束条件也会被质检打回

### 🔗 任务链管理要求
1. **依赖关系明确**：清晰定义任务间的依赖关系
2. **接口定义规范**：定义交付物之间的"接口"或"数据契约"
3. **可验证性要求**：要求交付物必须包含对前置任务的引用和整合说明
4. **追溯机制建立**：建立任务链追溯机制，确保可验证性

### 📋 合规性检查要点
1. **约束条件检查**：是否满足任务描述中的所有约束条件？
2. **可追溯性验证**：是否显式展示了与前置任务的关联？
3. **接口一致性**：是否遵循了任务间的交付物接口定义？
4. **依赖证明提供**：是否提供了可验证的依赖证明？
