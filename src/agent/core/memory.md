重要技术发现
agent_vm目录是老板电脑文件系统的挂载点
execute_command工具访问沙盒环境
filesystem工具访问老板电脑环境
动态工具需要通过call_dynamic_tool进行代理调用
write_text_file等工具有沙盒白名单限制
trading专家类型对复杂分析指令可能不稳定
简化的分析指令成功率更高
重新加载任务功能存在技术问题（TradingTask object has no attribute agent_state）
系统沙盒环境有时不可用
飞书CLI需要OAuth认证，需生成二维码供手机扫描
配置链接有时效性，需要及时处理
安装前需确认npm和Node.js环境
用户配置完成后需主动确认才能继续测试
飞书CLI配置流程：应用配置→用户登录→功能测试
文件上传：必须使用相对路径，不能使用绝对路径
表格创建：base创建后自动生成默认表"数据表"
字段创建：必须使用JSON格式，type参数需用字符串类型名
权限：用户登录后获得完整推荐权限，但某些API仍需特定权限
代理警告：系统检测到代理，但不影响功能使用
用户字段需要实际的open_id，不能使用"当前用户"等文本
删除操作需要--yes确认参数
飞书CLI工具功能完整，可用于自动化管理多维表格和云文档
表格创建、字段管理、记录操作等核心功能稳定可用

避坑经验
必须严格区分沙盒环境和老板环境
沙盒环境和老板电脑文件系统需要区分操作
动态工具不能直接调用，必须通过call_dynamic_tool
文件操作有权限限制，需要在白名单路径内操作
安装skill需要放在沙盒环境的skill文件夹里才能加载使用
使用npx skills命令搜索和安装技能
分析指令应简洁明确，避免过于复杂
创建新任务比重新加载更可靠
必须通过飞书主动发送消息同步进度
今日分析需重点关注短期市场因素
文件操作必须使用相对路径，不能使用绝对路径
