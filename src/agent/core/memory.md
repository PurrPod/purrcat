用户偏好与工作风格
用户是程序员老板，话少直接，喜欢干货，不喜欢客套话和官方话
用户偏好通过飞书接收详细分析结果和进度同步，需要主动发送简洁消息同步，但当前飞书连接可能有问题
用户喜欢将任务派发给小弟实现并发执行，关注实际功能效果而非理论描述
用户关注技术细节和工具可用性验证
用户关注信号与系统考试重点内容，已掌握第二章标准内容框架
用户需要处理学术PDF文档并整理笔记，偏好简洁但全面的笔记整理
用户需要将数学公式转换为Markdown LaTeX格式进行美化，要求保持原意不能加减字
用户喜欢使用trading专家进行专业投资分析

重要技术发现
agent_vm目录是老板电脑文件系统的挂载点，是沙盒环境
execute_command工具访问沙盒环境
filesystem工具访问老板电脑环境
动态工具需要通过call_dynamic_tool进行代理调用
write_text_file等工具有沙盒白名单限制
lark-cli安装成功，支持创建飞书多维表格
飞书CLI认证需要浏览器授权链接
简化的分析指令成功率更高
沙盒环境需要安装Python库来处理PDF
parse_document工具对某些PDF可能无效
可以使用strings命令初步查看PDF内容
需要区分扫描版PDF和文本版PDF的处理方法，扫描版PDF需要OCR处理才能提取文本
Remotion项目需确保有src/index.ts入口文件和package.json正确配置start脚本

避坑经验
沙盒环境和老板电脑文件系统需要区分操作
动态工具不能直接调用，必须通过call_dynamic_tool
文件操作有权限限制，需要在白名单路径内操作
分析指令应简洁明确，避免过于复杂，复杂任务要先加载对应skill
必须通过飞书主动发送消息同步进度，但需注意连接状态
lark-cli创建字段需要使用JSON格式，type参数为字符串
