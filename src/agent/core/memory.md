用户偏好与工作风格
用户偏好整洁有组织的笔记排版
用户需要解答技术细节问题
用户工作风格直接高效
用户偏好直接高效的测试方式
用户关注工具调用逻辑的正确性
用户需要区分沙盒环境和老板环境
用户需要将数学公式转换为Markdown LaTeX格式进行美化，要求保持原意不能加减字
用户偏好通过飞书接收分析结果，需要主动发送消息同步
用户关注股票投资分析，特别是A股和美股科技股
用户需要明确的今日交易建议，而非长期分析
用户喜欢使用trading专家进行专业投资分析
用户需要及时的任务状态更新

重要技术发现
agent_vm目录是老板电脑文件系统的挂载点
execute_command工具访问沙盒环境
filesystem工具访问老板电脑环境
动态工具需要通过call_dynamic_tool进行代理调用
write_text_file等工具有沙盒白名单限制
trading专家类型对复杂分析指令可能不稳定
简化的分析指令成功率更高
重新加载任务功能存在技术问题（TradingTask object has no attribute agent_state）
系统沙盒环境有时不可用

避坑经验
必须严格区分沙盒环境和老板环境
沙盒环境和老板电脑文件系统需要区分操作
动态工具不能直接调用，必须通过call_dynamic_tool
文件操作有权限限制，需要在白名单路径内操作
安装skill需要放在沙盒环境的skill文件夹里才能加载使用
使用npx skills命令搜索和安装技能
分析指令应简洁明确，避免过于复杂
创建新任务比重新加载更可靠
必须通过飞书主动发送消息同步进度
今日分析需重点关注短期市场因素
